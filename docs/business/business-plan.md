# ビジネス要件と競合調査

このドキュメントは、AILowcode スクールの toC 向け AI チャット SaaS における事業要件、実装時の必須方針、競合調査をまとめます。

## AIチャットSaaS 実装時の必須方針

README に記載しているサブスクリプションとクレジット仕様は、実装時に以下の方針で扱います。

### プランとクレジット

- 無料プランとプロプランを分ける
- 初回登録時に 10 クレジットを付与する
- チャット 1 回につき 1 クレジットを消費する
- プロプランは月額 980 円、年額 9,800 円を基本とする
- プロプランでは毎月 100 クレジットを付与する
- クレジット残高が不足している場合は、LLM Gateway を呼び出す前に処理を止める

### Stripe 実装

- Stripe Checkout / Billing Portal / webhook を使う場合、価格 ID と secret は環境変数で管理する
- webhook は署名検証を必須にする
- webhook event は重複配送される前提で、同一 event を複数回処理しても残高や subscription 状態が壊れないようにする
- subscription 状態は Stripe の表示だけに依存せず、アプリ DB に同期して認可や利用制限に使える形にする
- checkout 成功画面だけで有料化完了と判定せず、webhook による確定状態を基準にする

### AI / LLM Gateway 実装

- LLM Gateway の API key や provider 設定はサーバー側だけで扱う
- チャット実行前に認証、クレジット残高、プラン状態、入力サイズを検証する
- LLM Gateway 呼び出しに失敗した場合は、クレジット消費を確定させるか戻すかの方針を実装前に決め、DB 更新と一貫させる
- ユーザー入力、AI 応答、エラー情報の保存範囲を明確にし、不要な secret や個人情報をログに残さない

### Supabase 実装

- ユーザー別のチャット履歴、クレジット残高、決済状態を保存するテーブルは RLS を必須にする
- service role は Stripe webhook、管理処理、バッチ処理など、RLS をバイパスする必要があるサーバー側処理に限定する
- migration、RLS policy、型生成結果は同じ変更単位で更新する
- DB に金額、クレジット、subscription 状態を保存する場合は、監査しやすい履歴テーブルまたは event 記録を検討する

### リリース前確認

- `.env.example` に必要な環境変数名を追加し、実値は入れない
- Stripe test mode で checkout、webhook、月額、年額、キャンセル、期限切れを確認する
- LLM Gateway の成功、失敗、タイムアウト、クレジット不足を確認する
- `pnpm run lint`、`pnpm run build`、`pnpm exec opennextjs-cloudflare build`、必要に応じて `pnpm run preview` を通す

# 競合調査

## 結論

あなたが日本でAIスクールを開くなら、**「生成AIの使い方スクール」ではなく、明確に “AI業務アプリ開発スクール” に寄せるべき**です。

競合を見ると、主戦場は以下に分かれています。

| 領域                      | 主な競合              | 強い訴求                                        |
| ------------------------- | --------------------- | ----------------------------------------------- |
| 生成AIコミュニティ        | AI木曜会              | 心理的安全性、広いAI活用、月額制コミュニティ    |
| 女性向けキャリアスクール  | SHElikes              | 未経験・副業・転職・幅広いWebスキル             |
| プロダクト/PM教育メディア | Lenny’s Newsletter    | 高品質な知識、Slackコミュニティ、Product Pass   |
| エンジニア向けAI開発教育  | Matt Pocock / AI Hero | TypeScript × AIエンジニアリング                 |
| AI駆動開発コミュニティ    | AI駆動開発研究部      | Claude Code/Codex/Agent Skillなど最新AI開発実践 |
| Google Workspace業務改善  | Majincraft            | Google Workspace × Gemini × GAS の実務ハック    |

あなたの勝ち筋は、**「非エンジニアにも近い業務改善層」と「実装できる開発者層」の中間**です。
つまり、**“AIで業務アプリを作れる人材” を育てるポジション**が最も取りやすいです。

## 2. 各競合の分析

## AI木曜会

AI木曜会は、月額5,980円、入会金0円、年間契約不要を訴求している生成AIコミュニティです。公式ページ上でも「AIで稼ぐ」より「AIで人生を豊かにする」方向性や心理的安全性を打ち出しています。([AI木曜会｜生成AIを学び、実務で活かすコミュニティ][1])

noteでは、2024年5月開始の生成AI勉強コミュニティとして紹介され、オンライン勉強会、チャット、対面イベント、コワーキング的なつながり、100時間以上のアーカイブ動画などが示されています。([note（ノート）][2])

### 強み

- 月額5,980円で入りやすい
- 「怪しいAI稼ぎ」ではなく、健全なAI活用コミュニティとして見せている
- 初心者・会社員・教師・学生・子育て中の人など、幅広い層を包摂している
- コミュニティの温度感、心理的安全性、参加者インタビューが強い

### 弱み / 参入余地

AI木曜会は「AI活用コミュニティ」としては強いですが、**業務アプリを実際に設計・実装・納品できる人材育成**とは別物です。

あなたが勝つなら、以下を明確に分けるべきです。

> AI木曜会：AIを使いこなす人になる
> あなたのスクール：AIで業務アプリを作り、現場の業務を変えられる人になる

この違いはかなり大きいです。

## SHElikes

SHElikesは、女性向けキャリアスクールとして、Webデザイン、Webマーケティング、ライティングなどのデジタルスキルを幅広く学べるサービスです。公式サイトでは、50以上の職種スキルが定額で学び放題、最大70％還元の補助金制度を訴求しています。([shelikes.jp][3])

また、GMOとの提携リリースでは、オンライン動画、コーチング、ティーチング、コミュニティ、SHElikes内で挑戦できる仕事案件を提供していると説明されています。([GMOインターネットグループ株式会社][4])

### 強み

- 「未経験から働き方を変える」という大衆向け訴求が強い
- 女性向けキャリア支援のブランドがある
- Webデザイン・マーケ・ライティングなど、広い入口を持っている
- コーチング、コミュニティ、案件チャレンジがある
- 補助金訴求が強い

### 弱み / 参入余地

SHElikesは「広く学べる」ことが強みですが、反対に**AI業務アプリ開発に特化していない**点が空白です。

あなたはSHElikesと真正面から戦わない方がいいです。
女性向けキャリアスクール、Webデザイン副業、ライティング副業の市場に入ると、ブランド力・広告力・補助金対応で不利です。

狙うべきは、以下です。

> SHElikes：Webスキルを広く学んでキャリアを変える
> あなたのスクール：AI・ローコード・Supabaseで業務アプリを作れるようになる

## Lenny’s Newsletter

Lenny’s Newsletterは、プロダクト開発、グロース、キャリアに関する深いアドバイスを提供するニュースレター/Podcastです。公式Aboutでは、100万人以上の購読者、プロダクト・グロース・キャリアに関する深く調査された助言、30,000人以上の有料会員向けSlackコミュニティ、5年以上の過去記事、Product Passなどを打ち出しています。([レニーのニュースレター][5])

Product Passでは、年間購読者向けに多数のプレミアムプロダクト特典を提供しており、2026年4月時点の記事では25製品・30,000ドル超相当の価値を訴求しています。([レニーのニュースレター][6])

### 強み

- コンテンツ品質が非常に高い
- 「プロダクトを作る人」の知的ハブになっている
- Slackコミュニティが強い
- Product Passにより、購読料以上の特典価値を作っている
- 個人ブランドとメディア事業の完成度が高い

### 弱み / 参入余地

Lenny’s Newsletterは、スクールというより**高品質メディア＋コミュニティ＋特典バンドル**です。
直接競合ではありませんが、あなたが学ぶべき点は多いです。

特に参考にすべきは以下です。

- 無料コンテンツで信頼を作る
- 有料会員には過去アーカイブ・実践テンプレ・コミュニティを提供する
- ツール提供企業と組んで、会員特典を作る
- 「読むだけ」ではなく、キャリアと実務成果につながる導線を作る

あなたの場合、Product Passの代わりに以下が考えられます。

- Supabaseテンプレート
- Next.js業務アプリ雛形
- Nordcraftテンプレート
- 業務ヒアリングシート
- 要件定義テンプレート
- 顧客管理/予約/在庫/申請管理アプリのスターターキット
- Claude Code / Codex 用の開発ルール集
- Supabase RLS設計例
- SaaS導入チェックリスト

## Matt Pocock / AI Hero

AI Heroは、Matt PocockによるAIエンジニア教育ブランドです。公式サイトでは、AI時代でもソフトウェアエンジニアリングの基礎が重要であり、AI Heroは「出荷するコードに責任を持つ人」のためのものだと訴求しています。([aihero.dev][7])

また、AI SDK v6 Crash Courseは$149、Build Your Own AI Personal Assistant in TypeScriptの5日間コホートは$795と記載されています。([aihero.dev][8])

### 強み

- TypeScript教育の権威性がある
- AI SDK、Claude Code、AIエージェント、AI-ready codebaseなど、エンジニア向けの文脈が強い
- 教材の切り口が「AIでコードを書く」ではなく「AI時代に良いコードベースを作る」に寄っている
- AIエンジニアとAI-assisted developerの違いを明確にしている

### 弱み / 参入余地

AI Heroはかなり強いですが、日本市場では以下のギャップがあります。

- 英語教材
- エンジニア向け
- AI SDK / TypeScript中心
- 業務改善・SaaS導入・ローコード文脈は薄い
- 非エンジニアや業務部門には難しい

あなたが取るべきポジションは、Matt Pocockより少し実務寄り・業務改善寄りです。

> AI Hero：WebエンジニアをAIエンジニアにする
> あなたのスクール：業務改善人材をAI業務アプリ開発者にする

これは十分に差別化できます。

## AI駆動開発研究部

AI駆動開発研究部は、noteメンバーシップ形式のAI×開発コミュニティです。主宰者ページでは、月額9,800円、月1〜3回以上の本格勉強会、過去アーカイブ、Discordコミュニティ、ソースコード共有などが記載されています。([note（ノート）][9])

また、同ページでは、AI駆動開発研究部について「独学の限界を、最短で突破する」「AI×開発の実戦コミュニティ」と説明し、2026年5月時点で493名参加中と表示されています。([note（ノート）][9])

### 強み

- 月額9,800円でも成立している
- Claude Code、Codex、Agent Skill、マルチエージェントなど、最新テーマに強い
- 有料note、勉強会、Discord、コード共有の組み合わせが明確
- 「本職開発者の実践知」という訴求が強い

### 弱み / 参入余地

この競合は、あなたにとってかなり近いです。
ただし、AI駆動開発研究部は**開発者・個人開発者・AI駆動開発キャッチアップ層**に寄っています。

あなたは以下に寄せるべきです。

> AI駆動開発研究部：AI時代の開発トレンドと実装知見を追う
> あなたのスクール：現場課題をSupabase/Next.js/ローコードで業務アプリ化する

つまり、**「最新AI開発」ではなく「業務アプリ納品力」**に寄せる方が勝ちやすいです。

## Majincraft

Majincraftは、Google Workspace × Gemini活用の研究コミュニティです。noteのメンバーシップページでは、Google Workspace × Gemini 活用の研究コミュニティとして紹介されています。([note（ノート）][10])

個人向けプランは月額980円で、GeminiやGoogle Workspaceの活用術、オリジナルツール配布、まじん式の先行公開、掲示板対応、業務効率化相談などを提供しています。([note（ノート）][11])

さらに法人プランは月額9,800円で、1アカウント契約で全社員利用OK、個人プランの記事・ツールを社内研修やマニュアルとして共有・配布可能と訴求しています。([note（ノート）][12])

### 強み

- 月額980円で非常に入りやすい
- Google Workspace利用者に刺さる
- GAS、Gemini、Googleスライド、NotebookLMなど、業務現場に近い
- 「外部APIなし」「Google Workspace内完結」「セキュリティが厳しい企業でも使いやすい」という訴求が強い
- 法人プランの価格設計がうまい

### 弱み / 参入余地

Majincraftは、Google Workspace内の業務改善に強い一方で、以下はあなたの方が取りやすい領域です。

- Supabaseを使ったDB付き業務アプリ
- Next.js / Reactによる本格Webアプリ
- Nordcraftなどローコードフロントエンド
- 顧客管理、予約、在庫、申請、業務ダッシュボード
- SaaS導入後の現場業務フロー改善
- 受託開発レベルの要件定義・納品品質

Majincraftの強みは「Google Workspaceの中で完結」。
あなたの強みは「Google WorkspaceやSaaSで足りない部分を、業務アプリとして作れる」です。

## 3. 市場の空白

競合を見ると、次の空白があります。

## 空白1：AI活用と業務アプリ開発の間

多くのAIスクールは、ChatGPT活用、プロンプト、資料作成、業務効率化で止まりがちです。
一方、開発者向け教材は、AI SDK、Claude Code、RAG、エージェントなど技術寄りになります。

その中間に、以下の需要があります。

> 現場の業務課題を聞き、AIとローコードで小さな業務アプリに落とし込める人材

ここは、あなたの経歴と非常に相性が良いです。

---

## 空白2：Supabase特化の実務アプリ教育

日本市場では、Supabaseを「認証・DB・Storage・Edge Functions・RLS」まで含めて、業務アプリ開発に落とし込む教育はまだ薄いです。

Next.js、React、TypeScript単体の教材はあります。
AI活用教材もあります。
しかし、以下を一気通貫で教えるスクールは少ないです。

- 業務ヒアリング
- 要件定義
- DB設計
- Supabase Auth
- RLS
- Next.js画面実装
- AI機能追加
- 管理画面
- デプロイ
- 運用
- 顧客への納品

ここはかなり良いポジションです。

---

## 空白3：SaaS導入コンサル経験を活かしたAIスクール

あなたはSaaS導入コンサル5年以上という強みがあります。
これは単なる開発者との差別化になります。

AIスクール市場では、多くが「ツールの使い方」か「コードの書き方」に寄ります。
しかし実務では、以下が重要です。

- 既存業務の棚卸し
- SaaSで解決する部分と独自アプリで作る部分の切り分け
- 現場が使うUI設計
- 権限設計
- データ移行
- 運用フロー
- 導入後の定着

この実務感は、SHElikesやAI木曜会よりも深く、AI駆動開発研究部よりも業務現場寄りです。

## 6. カリキュラム案

## コアテーマ

**「AIで業務アプリを作る」ではなく、
「業務課題を見つけ、AIとローコードで運用可能なアプリにする」**

この順番が重要です。

## 8週間講座の例

| 週    | テーマ                          | 成果物                         |
| ----- | ------------------------------- | ------------------------------ |
| 1週目 | 業務課題の見つけ方              | 業務フロー図、課題リスト       |
| 2週目 | 要件定義と画面設計              | 要件定義書、画面ワイヤー       |
| 3週目 | DB設計 / Supabase基礎           | テーブル設計、RLS方針          |
| 4週目 | 認証・権限・管理画面            | ログイン付き業務アプリ         |

## 8. 競合に対する勝ち筋

## 勝ち筋1：ローコードを前面に出す

あなたの強みは、ガチガチのフルスクラッチだけではありません。
**ローコードと実装の境界を扱えること**です。

そこで、以下の橋渡しができます。

> ノーコードでは足りない。
> でも、全部フルスクラッチする必要もない。
> Supabase・Nordcraft・Next.js・AIを組み合わせて、現場で使える業務アプリを作る。

このメッセージはかなり刺さります。

## 勝ち筋2：受託開発50件以上の実績を教材化する

受託開発50件以上は、単なる肩書きではなく、教材の源泉です。

競合と差別化するには、以下を出すべきです。

- よくある業務アプリ要件10選
- 失敗する業務アプリの特徴
- 顧客ヒアリング質問集
- 見積もりの作り方
- 要件定義テンプレート
- 納品前チェックリスト
- Supabase RLS設計パターン
- 使われない管理画面の特徴
- SaaSで済ませるべき業務 / 作るべき業務の切り分け

これは、AI活用だけを教える競合には出しづらい価値です。

## 勝ち筋3：SaaS導入コンサルの視点を入れる

「AIアプリを作ろう」だけだと、開発者スクールになります。
あなたはそこに、SaaS導入コンサルの視点を入れるべきです。

例：

- Notionで十分な業務
- Google Workspaceで十分な業務
- kintoneで十分な業務
- Supabaseで作るべき業務
- Next.jsで作るべき業務
- AIエージェント化すべき業務
- そもそも自動化しない方がいい業務

この判断軸を教えられると、単なる開発スクールではなく、**業務改善人材育成スクール**になります。
