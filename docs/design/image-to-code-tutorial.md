## 1. 目的

本文書は、ChatGPT Images 2.0 および Codex を用いた Image-to-Code ワークフローの具体的な実装手順を定義する。原則・背景については「Image-to-Code フロントエンドワークフロー標準（DEP-WF-001）」を参照すること。

## 2. 対象読者

- Image-to-Code ワークフローを初めて実践するフロントエンドエンジニア
- デザインエンジニア

## 3. 前提知識

- ChatGPT（Images 2.0）の基本操作
- Codex（または同等のコーディングエージェント）の基本操作
- React + Next.js の基礎

## 5. 手順

### 5.1 ウェブサイト画像の生成

#### 5.1.1 画像生成スキルの読み込み

1. [imagegen-frontend-web SKILL.md](https://github.com/Leonxlnx/taste-skill/blob/main/skills/imagegen-frontend-web/SKILL.md) の内容をコピーする
2. ChatGPT のプロンプトに貼り付ける、または Markdown ファイルとして添付する

#### 5.1.2 プロンプトの実行

以下の構成でプロンプトを作成する：

- スキル使用の指示
- プロダクト・業界の指定
- セクション一覧（1 セクション = 1 画像）
- スタイル方向性の具体的指定
- 禁止事項（ありきたりな SaaS ビジュアル、グラデーション blob 等）

**プロンプトテンプレート**：

```
上記のスキルに基づいて、[プロダクト/業界]のウェブサイトの画像を生成してください。
デザインは[N]個のセクションで構成し、1セクションにつき1枚、合計[N]枚の画像を生成してください。
[スタイル方向性の具体的記述]
メインカラーは[色]を使用してください。すべての画像は横向きで。
```

#### 5.1.3 品質向上のヒント

1. 複数のチャットで同一プロンプトを実行し、比較選定する
2. Thinking モードを使用すると高品質な出力が得られやすい
3. 1 セクション = 1 画像の原則を守る
4. Images 2.0 は 1 回で最大 10 枚まで生成可能

### 5.2 画像のブラッシュアップ

生成結果に対して以下の調整が可能である：

- **単一画像の編集**：特定の 1 枚を指定して修正を指示する
- **全画像のスタイル変更**：全セクションのスタイルを一括で変更する

満足のいく結果が得られたら、全画像をダウンロードする。

### 5.3 アセットの抽出

#### 5.3.1 抽出対象

以下の要素を個別アセットとして抽出する：

- プロダクト写真
- 抽象 3D 形状
- イラスト
- アイコン
- 背景テクスチャ
- デバイスモックアップ

#### 5.3.2 抽出プロンプト

新規チャットでセクション画像をアップロードし、以下の形式で指示する：

```
[画像内の位置/説明]にある[対象]を抽出し、生成してください。
合計[N]枚の抽出・生成画像を作成してください。
画像は変更せず、まったく同じ見た目にしてください。
```

**重要**：プロンプトには必ず「生成して抽出」を含めること。「抽出」のみだと単純な切り抜き処理になる。

**複数アセットの抽出例**：

```
この画像には4つの画像が使われています。それらを抽出し、4つすべてを生成してください。
合計4枚の異なる画像を生成してください。
```

### 5.4 背景の削除

ChatGPT には透過画像の生成機能がないため、必要に応じて以下のツールで背景を削除する：

- [Adobe Express 背景削除](https://express.adobe.com/home/tools/remove-background)（無料）

### 5.5 コードへの変換

#### 5.5.1 使用ツール

- Codex（推奨）
- その他、画像読み取りとプロジェクト編集が可能なコーディングエージェント

#### 5.5.2 基本ルール

- 毎回 1 セクションずつ作業する
- ウェブサイト全体を一度に変換しようとしない

#### 5.5.3 ヒーローセクションの実装

以下のプロンプトテンプレートを使用する：

```
ウェブサイトを作成します。画像はこちらで提供するので、
画像内のアセットは生成・構築せず、コンポーネントと細部に集中してください。
これがヒーローセクションです。これを複製してください。
その後、画像を配置するフォルダのディレクトリを提示してください。
React + Next.js を使用し、ローカル開発サーバーを起動してください。
```

#### 5.5.4 アセットの配置

1. Codex が指定したフォルダに、抽出したアセット画像を配置する
2. サイトを更新するか、画像追加を Codex に通知する
3. コンポーネントのサイズ・位置を調整しブラッシュアップする

#### 5.5.5 残セクションの実装

同じ手順を全セクションで繰り返す。配置ずれや要素の重なりが発生した場合は、スクリーンショットを撮影して Codex にフィードバックし修正させる。

**注意**：1 回のプロンプトで完璧な結果は期待しないこと。反復的な調整が前提である。

### 5.6 アニメーションの追加

全セクションの静的実装が完了した後、以下を追加する：

1. **スクロールアニメーション**
2. **ホバーアニメーション**
3. **レスポンシブ最終調整**

**プロンプト例**：

```
新しいセクションにスクロール進入した際のアニメーションを追加してください。
スムーズでクリーンな印象にすること。
```

| アセット抽出            | 「[位置]の[対象]を生成して抽出。単体・クリーン・高解像度で」                                                         |
| コード変換（初回）      | 「画像のアセットは生成せず、コンポーネントと細部を複製。使用フォルダを提示。React + Next.js + ローカルサーバー起動」 |
| コード変換（2回目以降） | 「このセクション画像を複製。アセットファイル名は[一覧]」                                                             |
| 磨き込み                | 「[要素]が[状態]。位置を[方向]に移動し、サイズを[数値]% [拡大/縮小]。[基準要素]に合わせる」                          |
| アニメーション追加      | 「スクロール進入時のアニメーションを追加。スムーズかつクリーンに」                                                   |

## 7. 最終成果物のチェックリスト

- [ ] 全セクションがリファレンス画像と一致しているか
- [ ] スクロールアニメーションが全セクションに適用されているか
- [ ] ホバーアニメーションがインタラクティブ要素に適用されているか
- [ ] デスクトップ / タブレット / モバイルでの表示崩れがないか
- [ ] アセット画像の解像度・品質は十分か
- [ ] カラー・タイポグラフィ・ボタンスタイルに一貫性があるか
