# テスト戦略

AILowcode プロジェクトにおけるテストの方針と、機能実装時に満たすべき検証ルールを定めます。

このリポジトリの初期状態では、テストランナーと E2E テスト環境はまだ導入していません。現時点で常時実行する検証は `pnpm run lint`、`pnpm run build`、`pnpm exec opennextjs-cloudflare build` です。

以下のテスト種別は、対象機能を実装するタイミングで追加する必須方針として扱います。未導入のツールを使う場合は、`package.json`、CI、README、関連 docs を同じ変更で更新してください。

## ドキュメント一覧

| No  | ドキュメント             | 内容                                       |
| --- | ------------------------ | ------------------------------------------ |
| 01  | 概要（本ドキュメント）   | テスト全体戦略                             |

| 05  | 手動テストチェックリスト | リリース前の確認項目                       |

## テストレベルの考え方

- **単体テスト**: クレジット消費、サブスクリプション状態判定、入力検証、外部サービスレスポンス変換など、分岐のあるロジックを検証する
- **コンポーネントテスト**: 主要な UI コンポーネントの表示、入力、エラー状態、ローディング状態を検証する
- **E2E テスト**: サインアップ、ログイン、チャット送信、クレジット不足、決済、プラン変更など、ユーザー影響の大きいフローを検証する

## ツール

| カテゴリ                | 方針                                                     |
| ----------------------- | -------------------------------------------------------- |
| 単体テスト              | 導入時は Vitest を第一候補にする                         |
| UI コンポーネントテスト | 導入時は React Testing Library + Vitest を第一候補にする |
| E2E テスト              | 導入時は Playwright を第一候補にする                     |
| カバレッジ              | 導入時は v8 coverage を第一候補にする                    |

## 実装時の必須方針

- 新しい `test` 系スクリプトを追加した場合は、CI の実行項目にも追加する
- Stripe、LLM Gateway、Supabase Auth、RLS、クレジット残高に関わる変更は、正常系だけでなく失敗系と権限エラーも検証する
- テスト未導入の段階であっても、PR 前に `pnpm run lint`、`pnpm run build`、`pnpm exec opennextjs-cloudflare build` を実行する
- Cloudflare Workers 上で挙動差が出やすい機能は、`pnpm run preview` で runtime 差分を確認する
