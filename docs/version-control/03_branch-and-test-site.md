# ブランチとテストサイト

## ブランチ方針

このプロジェクトでは、**`main` を唯一の基準ブランチ**として扱います。

- 本番と同じ状態のブランチは `main` とする
- `dev` などの長期運用ブランチは作成しない
- すべての変更は `main` から作業ブランチを切り、PR 経由で `main` にマージする

---

## テスト環境の方針

このプロジェクトでは、**Cloudflare Workersのローカルプレビューをテスト環境の標準**とします。

### 仕組み

- 作業ブランチで `pnpm run preview` を実行し、Workers ランタイム上で確認する
- レビュアーはその確認結果をもとにブラウザ確認を行う
- PR がマージされたら `pnpm run deploy` で Cloudflare Workers に反映する

```text
作業ブランチ push
  → PR 作成
  → pnpm run preview で Workers ランタイム確認
  → ブラウザ確認・レビュー
  → main にマージ
  → pnpm run deploy で本番反映
```

### この方針を採用する理由

- `dev` ブランチなどの長期運用ブランチが不要になる
- PR 単位で本番に近いランタイム確認ができる
- 追加のブランチ管理コストが発生しない
- main シングルブランチ運用との相性が最も良い

---

## 補足

本番環境は `main` へのマージ後に Cloudflare Workers へのデプロイで反映されます。

ローカルプレビューは `pnpm run preview` で確認できます。
